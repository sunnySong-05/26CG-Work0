# 26CG-Work0
计算机图形学上机实验-Work0-万有引力粒子群仿真
#项目介绍
这是一个基于Taichi高性能计算语言开发的GPU加速的粒子群交互模拟系统。程序模拟了10000个粒子在鼠标引力作用下的群体运动，利用GPU并行计算实现流畅的实时交互效果。我们只需移动鼠标，即可看到粒子群如同被磁力吸引般跟随流动。
##项目架构——src布局
###目录结构
src/
├── Work0/
│   ├── config.py
│   ├── physics.py
│   ├── main.py
│   ├── README.md

src布局相较于扁平布局具有显著优势。核心逻辑代码整齐地放置在src/Work0/目录下，而项目根目录则用于存放pyproject.toml、README.md等配置文件，实现了代码与配置的清晰分离，避免了混合存储可能导致的配置错误。
此外，src布局还有助于保持导入路径的纯净。项目根目录下的pyproject.toml、README.md、uv.lock等配置文件在扁平布局中可能被视为Python模块，造成混淆。但在src布局中，只有src/目录下的内容才会被视为可导入的包，根目录的配置文件被完美隔离，从而确保了可编辑安装时导入路径中只包含真正的代码文件。

##代码逻辑

###特殊文件_init_.py
这一特殊文件用于将项目中的不同模块导入到主程序中，使Python把Work0文件夹识别为一个标准的模块包，从而允许我们在文件之间互相 import。这样可以方便地在主程序中使用这些模块，而不需要直接从每个模块中导入。

###配置模块config.py
配置模块负责把所有硬编码的参数抽离出来集中管理。这种设计体现了关注点分离的原则，将参数定义与核心逻辑解耦，使得后续调试和优化变得更加便捷。
在物理系统参数方面，模块定义了粒子总数、引力强度、空气阻力系数和边界反弹系数。其中粒子总数决定计算负载，引力强度控制鼠标对粒子群的影响程度，空气阻力系数模拟能量耗散过程，边界反弹系数则定义了粒子碰撞窗口边缘时的能量损失比例。渲染系统参数则负责视觉呈现效果，包括窗口分辨率、粒子绘制半径和粒子颜色。

###物理系统模块physics.py
物理引擎模块是整个系统的计算核心，只包含Taichi的数据结构定义和`@ti.kernel`修饰的底层并行算子，与窗口绘制无关，只专注于数学计算。
在数据结构层面，模块在GPU显存中创建了两个向量字段，分别存储所有粒子的位置和速度信息，避免了CPU与GPU之间的频繁数据传输。
模块通过`@ti.kernel`定义了初始化与物理更新两个并行算子。初始化算子为数万个粒子同时赋予随机的初始位置；物理更新算子则定义了每一帧的行为逻辑——根据鼠标位置计算引力，施加空气阻力使运动衰减，更新位置并进行边界碰撞检测。这一连串计算在GPU上完全并行执行，数万个粒子的物理状态可以同时更新，使系统能够保持流畅运行。

###主程序模块main.py
主程序模块负责初始化运行环境、读取用户鼠标输入，并将计算好的粒子位置数据渲染到屏幕上。它扮演着连接配置参数、物理逻辑和图形界面的桥梁角色——从`config.py`读取窗口尺寸和粒子颜色等设置，调用`physics.py`中的GPU并行算子完成物理更新，最后将计算结果绘制为动态画面呈现在用户面前。这一模块的存在使得各个组件能够各司其职，形成了清晰的关注点分离。

##实现功能

GPU加速的大规模粒子模拟是本程序的基础能力，通过Taichi框架将物理计算编译为GPU内核代码，程序能够同时模拟数万个粒子的运动状态。当用户在窗口内移动鼠标时，所有粒子都会受到指向鼠标位置的引力作用，形成跟随流动的视觉效果。程序综合运用了引力加速、空气阻力衰减和边界反弹三种基础物理模型，三者叠加形成了自然流畅的群体运动特征。
